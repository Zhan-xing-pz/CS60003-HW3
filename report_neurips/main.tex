\begin{abstract}
本文完成深度学习与空间智能 HW3 的两个实验任务。题目一构建真实多视角重建、文本到 3D、单图到 3D 与背景场景重建的完整链路：物体 A 由手机环绕视频经 COLMAP 和 3D Gaussian Splatting (3DGS) 重建，物体 B 由 DreamFusion/SDS 根据文本生成，物体 C 由 Stable Zero123 根据单张图像生成，背景采用 Mip-NeRF 360 Flowers 场景；随后将 mesh 表面采样为显式点云，与 3DGS 资产在统一坐标系中完成融合预览。物体 A 的测试指标达到 SSIM 0.9222、PSNR 24.2401 和 LPIPS 0.1431。题目二使用 LeRobot ACT 研究 CALVIN 的跨环境策略泛化，在相同网络结构和超参数下比较 A-only 与 A/B/C 联合训练，并仅使用训练域 validation split 选择最优 checkpoint。环境 D 的 200-batch zero-shot 评估显示，联合模型的 mean \actlone{} 为 0.4471，优于 A-only 的 0.4963。两项实验共同展示了显式 3D 表示、多源生成先验与时序动作建模在空间智能任务中的能力，同时也揭示了短训练、表示转换和视觉分布偏移带来的限制。
\end{abstract}

\section{题目一：基于 3DGS 与 AIGC 的多源资产生成与真实场景融合}

\subsection{任务背景与总体流程}
题目一要求完成一条从真实采集、生成式 3D 建模到统一场景融合的全链路。实验准备三类来源不同的物体资产：真实多视角重建物体 A、文本生成物体 B 和单图生成物体 C；同时从公开数据集重建真实背景，并将四类结果放入同一三维坐标系生成漫游预览。该任务的关键困难不只在于单个模型的训练，还在于相机位姿质量、不同表示形式之间的转换、物体尺度与坐标对齐，以及生成资产在未观测视角下的几何一致性。

本实验采用“显式点/高斯中间层”统一表示。真实物体和背景由 COLMAP 位姿初始化后训练 3DGS；DreamFusion 与 Zero123 输出的隐式场或 mesh 则在表面采样为带颜色点云。所有资产经过尺度、偏航角和平移变换后进入同一坐标系，由轻量 point-splatting renderer 生成 1280$\times$720、30 FPS、180 帧的融合预览。图~\ref{fig:t1_overview} 展示输入资产与融合原型。

\subsection{数据与资产准备}
\paragraph{物体 A：真实多视角数据。}
原始手机视频包含 1370 帧，分辨率为 1280$\times$720、帧率为 30 FPS。实验均匀抽取 72 帧并将最大宽度缩放至 1024，用于 SIFT 特征提取与 COLMAP 增量式 SfM。由于所有帧来自同一手机相机，设置 \texttt{single\_camera=1} 共享内参，以减少逐帧估计内参造成的漂移。

\paragraph{物体 B：文本条件。}
文本 prompt 为 ``a small jade dragon statue with translucent green material, intricate carved scales, and a polished museum-object look''。该物体与真实杯子在语义和几何上明显不同，便于观察纯生成资产在融合场景中的形状自由度与多视角一致性。

\paragraph{物体 C：单图条件。}
输入为一张 1448$\times$1086 的真实杯子照片。首先使用 GrabCut 得到前景 alpha 和 RGBA 图像，再将其作为 Stable Zero123 的条件视图。单图输入能较好约束正面外观，但背面形状只能依赖预训练先验补全。

\paragraph{背景场景。}
背景选用 Mip-NeRF 360 的 Flowers 场景，包含 173 张多视角图像、官方 COLMAP sparse 模型和多级下采样图像。它具有草地、叶片和花簇等高频纹理，适合检验有限迭代数下大范围 3DGS 的重建能力。

\subsection{方法原理与实现}
\subsubsection{COLMAP 与 3D Gaussian Splatting}
物体 A 依次执行 SIFT GPU 特征提取、穷举匹配、增量式 SfM 和图像去畸变。最大 sparse model 成功注册 49 张图像，包含 2086 个稀疏点、11424 个观测，平均 track length 为 5.4765，平均重投影误差为 0.6865 px。图~\ref{fig:t1_colmap} 给出质量统计。

\begin{figure}[htbp]
\centering
\includegraphics[width=0.9\linewidth]{figures/topic1_object_a_colmap_quality.png}
\caption{物体 A 的 COLMAP 注册规模和 sparse model 质量指标。}
\label{fig:t1_colmap}
\end{figure}

3DGS 使用一组可学习的各向异性高斯表示场景。每个高斯包含位置、协方差、不透明度和视角相关颜色；渲染时将高斯投影到图像平面并按深度顺序进行 alpha compositing：
\begin{equation}
  C(\mathbf{p})=\sum_i T_i\,\alpha_i\,c_i,\qquad
  T_i=\prod_{j<i}(1-\alpha_j).
\end{equation}
训练通过渲染图像与真实图像之间的像素误差和结构相似性损失联合优化参数，并周期性执行高斯增密和裁剪。物体 A 与 Flowers 背景均训练 7000 iterations，并保留测试视角用于 PSNR、SSIM 和 LPIPS 评估。

\subsubsection{DreamFusion 文本到 3D}
物体 B 使用 threestudio 的 DreamFusion 流程。随机相机下渲染的图像被送入 Stable Diffusion v1.5，利用 Score Distillation Sampling (SDS) 将文本先验蒸馏到三维表示。其梯度可概括为
\begin{equation}
  \nabla_{\theta}\mathcal{L}_{\mathrm{SDS}}
  \approx \mathbb{E}_{t,\epsilon}\left[
  w(t)(\hat{\epsilon}_{\phi}(x_t;y,t)-\epsilon)
  \frac{\partial x}{\partial\theta}\right],
\end{equation}
其中 $\theta$ 为三维场参数，$y$ 为文本条件。为适配本机 RTX 5070 Ti 的 CUDA 兼容性，几何编码使用纯 PyTorch 的 \texttt{ProgressiveBandFrequency} 与 \texttt{VanillaMLP}，训练 1200 steps。针对 Windows 下原生 marching cubes 导出易卡住的问题，额外实现 checkpoint 直读脚本，仅加载 \texttt{geometry.*} 权重，采样密度场后导出 OBJ。

\subsubsection{Stable Zero123 单图到 3D}
物体 C 使用 Stable Zero123 将条件图像、相对相机方位和扩散先验结合，优化可从新视角渲染的三维表示。训练共 600 steps，分辨率按 128、192、256 逐级提升，随机相机 batch size 按 4、2、1 递减，以控制显存占用。最终导出 OBJ mesh 和 50000 点表面采样。与文本生成相比，单图条件能够更强地约束输入视角纹理，但不可见区域存在形状收缩和先验幻觉。

\subsubsection{统一表示与场景融合}
3DGS 与 AIGC 资产的原生表示并不一致：前者是带协方差和透明度的高斯集合，后者通常为 NeRF/隐式场或 mesh。本实验将 AIGC mesh 表面采样为 $(x,y,z,r,g,b)$ 点集，并用
\begin{equation}
  \mathbf{x}'=sR_z(\psi)\mathbf{x}+\mathbf{t}
\end{equation}
完成尺度、偏航与平移对齐。预览阶段直接进行点 splatting；更高质量的后续方案可将采样点初始化为 Gaussian primitives，或在 Blender 中将 mesh 与背景渲染结果组合。

\subsection{现象分析}
\paragraph{多视角真实重建的几何一致性最好。}
物体 A 的 SSIM 和 LPIPS 明显优于大范围背景，且 GT 与渲染图在轮廓和视角变化上高度一致。这来自真实多视角像素监督和可靠相机位姿。其主要瓶颈不是生成先验，而是低纹理陶瓷、高光反射与 COLMAP 注册覆盖率。

\paragraph{文本生成自由度最高，但几何约束最弱。}
DreamFusion 可以在没有真实图像的情况下生成语义匹配的玉龙资产，但 SDS 只通过二维扩散先验间接约束三维场，容易出现 Janus、多面不一致和漂浮密度。1200 steps 的快速配置足以验证流程，却不足以获得精细拓扑。

\paragraph{单图生成更忠实于输入外观，但不可见区域具有不确定性。}
Zero123 对正面颜色和类别保留较好，同时必须依赖先验推断背面。物体 C 的结果说明单图方法位于真实多视角重建和纯文本生成之间：条件更强、自由度更低，但无法消除背面幻觉和尺度歧义。

\paragraph{统一表示带来可组合性，也损失原生信息。}
将 mesh 采样为点云后，三类资产可以使用同一变换和渲染接口，工程实现简单；但这一转换丢失了 mesh 拓扑、材质参数，也没有保留 3DGS 的各向异性协方差与透明度。图~\ref{fig:t1_compare} 的相对评分概括了三条路线在几何、纹理和计算开销上的差异。

\subsection{小结}
题目一完成了物体 A 的 COLMAP 与 3DGS 重建、物体 B 的 DreamFusion 文本生成、物体 C 的 Stable Zero123 单图生成、Flowers 背景 3DGS 训练，以及统一点云场景的融合预览。实验表明，多视角真实监督具有最佳几何一致性，文本生成具有最高语义自由度，单图生成则在输入忠实度和未见区域不确定性之间折中。后续提升方向包括将 Flowers 训练至 30000 iterations、为 B/C 增加训练步数与更高分辨率，以及使用支持高斯与 mesh 联合光栅化的渲染器改善遮挡、光照和材质一致性。

\section{题目二：基于 LeRobot 的 ACT 策略跨环境泛化挑战}

\subsection{任务背景}
机器人模仿学习通常需要从视觉观测、机器人状态与专家动作序列中学习闭环控制策略。然而，当训练和测试环境在背景纹理、光照、物体摆放或相机观测分布上发生偏移时，视觉策略容易出现跨环境泛化退化。本实验关注的核心问题是：仅在单一环境中训练的 ACT 策略能否泛化到未见过环境，以及混合多个训练环境是否能提升 zero-shot 跨环境表现。

题目要求完成三个部分：首先，基于 LeRobot 框架中的 ACT 算法，仅使用 CALVIN 环境 A 训练基础视觉-动作策略；其次，将环境 A/B/C 数据混合，在相同 ACT 网络结构和超参数下训练多环境联合模型；最后，将两个模型部署到完全未见过的环境 D 中进行 zero-shot 测试，并比较成功率或动作误差。由于本次提供的 D 环境离线数据不包含真实任务完成信号，本文使用 mean \actlone{} 作为主要评估指标，success rate 保持空值。

\subsection{数据集描述}
实验使用 Hugging Face 上的 \texttt{huiwon/calvin\_task\_ABC\_D} 数据集，并转换为 LeRobot v3.0 数据格式。CALVIN 数据集包含多个视觉环境，每个环境提供机器人状态、RGB 观测和专家动作。本文将环境 A 作为单环境训练集，将环境 A/B/C 合并为联合训练集，并将环境 D 完全保留为最终 zero-shot 测试环境。

训练和验证不使用环境 D。对于 \methodA{}，从环境 A 中按 episode 做 90\%/10\% 划分；对于 \methodABC{}，分别从 A、B、C 各自按 90\%/10\% 划分后混合得到训练与验证集合。这样可以保证 best checkpoint 的选择只依赖训练域内部的 validation \actlone{}，避免用测试环境 D 调参。

\subsection{方法原理}
ACT 是一种面向精细操作任务的轻量级模仿学习策略。根据 LeRobot 文档，ACT 使用视觉骨干网络处理多视角图像，通过 Transformer 编码器融合图像特征、机器人状态和潜变量，并由 Transformer 解码器一次性预测未来 $k$ 个动作组成的动作块。本文设定 chunk size 为 16，即模型每次输出 16 步未来动作。

动作分块机制的关键优势在于时间一致性。普通逐步预测策略需要在每一帧都独立决定下一动作，因而容易对视觉噪声或单帧分布偏移敏感；ACT 通过输出连续动作序列，使策略在短时间窗口内具有更平滑的控制意图。面对环境 D 中未见过的视觉外观时，这种机制有助于降低短时动作抖动，但如果视觉编码器只见过单一环境，仍可能把背景或光照变化误解释为状态变化。因此，本文假设 A/B/C 联合训练能够提升视觉表征的环境不变性，并与 ACT 的动作分块机制互补。

\subsection{现象分析}
\paragraph{多环境联合训练显著提升跨环境泛化。}
最终 D 环境 mean \actlone{} 中，\methodABC{} 为 0.4471，显著低于 \methodA{} 的 0.4963，下降约 9.9\%。这表明训练时引入 A/B/C 多环境视觉变化后，模型在面对未见过的 D 环境时能产生更接近专家的动作。该结果符合跨环境泛化的直觉：单环境 A-only 模型容易学习到与环境 A 外观绑定的视觉特征，而 ABC 联合训练迫使策略从多种背景和布局中提取更稳定的任务相关线索。

\paragraph{训练 loss 与泛化指标并不同步。}
在 30000-step 训练中，A-only 的训练 loss 仍然下降，但 validation \actlone{} 在 14000 step 后没有持续改善。类似地，ABC 在 27000 step 达到最好 validation \actlone{} 后，30000 step 反而回升到 0.4502。这说明更长训练不必然带来更好泛化，尤其在视觉模仿学习中，模型可能继续拟合训练域中的细节，却不提升验证域和测试域表现。因此，本实验使用 validation \actlone{} 而非最后一步选择 checkpoint 是必要的。

\paragraph{ACT 动作分块的鲁棒性与限制。}
ACT 的动作分块机制一次预测多个未来动作，能够在短时窗口内平滑控制输出，减少逐帧视觉噪声对动作的直接影响。在环境 D 的 visual distribution shift 下，这种机制有助于保持动作序列的连续性。但实验也显示，动作分块不能完全替代视觉泛化能力：A-only 即使使用 ACT，在 D 环境中仍明显落后于 ABC。这说明 ACT 的 temporal smoothing 与多环境视觉数据具有互补关系，前者提升短期动作一致性，后者提升跨环境表征鲁棒性。

\paragraph{30000-step 设置的选择。}
探索过程中比较了 25000 与 30000 steps。30000 steps 对 \methodABC{} 有小幅收益：D mean \actlone{} 从 0.4522 降至 0.4471；但对 \methodA{} 无收益，D mean \actlone{} 从 0.4918 升至 0.4963。综合题目关注的跨环境泛化对比，30000 steps 下 ABC 与 A-only 的差距更清晰，同时 validation-best 机制避免了直接使用过拟合的最后一步 checkpoint，因此本文采用 30000 steps 作为最终设置。

\subsection{结论}
本文完成了基于 LeRobot ACT 的跨环境泛化实验。结果表明，在相同网络结构与超参数下，A/B/C 多环境联合训练相比单环境 A-only 训练显著降低了环境 D 上的 zero-shot 动作误差。该现象说明，多环境数据能够提升视觉策略对环境外观变化的鲁棒性；ACT 的动作分块机制进一步提供了时序一致的动作输出，但其效果依赖于视觉表征是否足够环境不变。未来可进一步加入真实在线 rollout success rate、更多环境随机化和更长 horizon 的任务级评估。
